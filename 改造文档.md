## 项目建设
请用 Docusaurus classic 模板创建一个文档站项目 minner-fun（TypeScript），并完成这些工作：

1) 配置站点信息：title=Minner，tagline=Smart Contract Notes · Protocol Research · Builder Log。
2) 开启 i18n：defaultLocale=en，locales=[en, zh-Hans]，navbar 添加 localeDropdown。
3) 设计 docs 内容结构：Notes / Protocols / Projects 三大栏目，并用 sidebars.ts 以 category + autogenerated 实现。
4) 创建示例文档（英文+中文镜像各 1-2 篇）：
   - notes/foundry/intro.md
   - notes/foundry/cheatcodes.md
   - protocols/uniswap-v2/overview.md
   - projects/fuzzing-template.md
   中文版本放在 i18n/zh-Hans/docusaurus-plugin-content-docs/current/ 对应路径。
5) 首页改成简洁技术名片风格：三块入口按钮（Notes/Protocols/Projects）+ GitHub 链接。
6) 确保 npm run start/build 正常，语言切换后路径为 /zh-Hans/ 前缀且文档可访问。
（依旧部署到 vercel）

## 优化升级
可以更有内容感，但保持简洁即可。几个小步能立刻改善：
首屏信息密度：在标题下加一句更具体的价值说明（面向谁、能得到什么），再补一个主 CTA（“开始阅读”指向 Notes 入口）+ 次级 CTA（“在 GitHub 查看”）。
视觉层次：加一个淡雅渐变/网格背景或简单插画，配合分割线让 hero 区与底部区隔，不会空旷。
导航聚焦：将 Notes/Protocols/Projects 的按钮改成“卡片”式展示，带一句摘要和最近更新的文档标题/日期。
内容入口：在首页再加一行“最近更新”或“精选”列表（3–6 条），让用户直接能点进热门或新文档。
国际化提示：在 hero 区放一个小的中/英切换提示或文案，弱化找不到中文内容的担忧。
Footer 丰富度：加上联系邮箱/社交、贡献指南或“提出 Issue”链接，让底部不空。
如果想要更强的品牌感，再加入：
自定义 favicon / OpenGraph 卡片与主题色一致。
小图标集成到导航或卡片（保持轻量）。
可以先选 2–3 个快改项（背景+CTA+精选列表）上线看看效果，再逐步迭代。需要的话，我可以直接改首页 src/pages/index.tsx 和样式给你一个更丰富的版本。
