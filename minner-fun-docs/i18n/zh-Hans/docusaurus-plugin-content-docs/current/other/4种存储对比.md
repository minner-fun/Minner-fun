# Solidity 中的 4 种存储类型详解

## 📊 完整对比表

| 存储类型 | 生命周期 | 编程类比 | Gas消耗 | 可修改 | 使用场景 |
|---------|---------|---------|---------|--------|---------|
| **storage** | 永久存在 | 💾 数据库 | 💰💰💰 最贵 (5000-20000 gas) | ✅ | 合约状态变量 |
| **transient storage** | 交易期间 | 🔄 Redis/交易级缓存 | ⚡ 很便宜 (~100 gas) | ✅ | 跨函数临时数据传递 |
| **memory** | 函数调用期间 | 🧠 函数局部变量 | 💰 适中 | ✅ | 函数内临时数据 |
| **calldata** | 函数调用期间 | 📨 函数参数(const) | ⚡ 最便宜 | ❌ 只读 | 外部函数参数 |

## 🔍 详细说明

### 1️⃣ Storage（永久存储）

**类比：数据库**
- 程序重启后依然存在
- 数据永久保存在区块链上
- 最昂贵的存储方式

```solidity
contract Example {
    // storage: 状态变量，永久存储
    uint public storageValue;
    mapping(address => uint) public balances;
    
    function updateStorage() external {
        storageValue = 100;  // 写入永久存储，贵！
        // 下次调用还能读到这个值
    }
}
```

**特点：**
- ✅ 永久保存
- ✅ 所有交易都能访问
- ❌ Gas 消耗高
- ❌ 需要谨慎管理（避免存储膨胀）

---

### 2️⃣ Transient Storage（瞬态存储）

**类比：Redis/交易级缓存**
- 交易结束后自动清除
- 可以在同一交易的多个函数调用之间传递数据
- EIP-1153 引入，Solidity 0.8.24+ 支持

```solidity
contract Example {
    function createPool(uint premineAmount) external {
        // 存储：在整个交易期间有效
        assembly { tstore(0, premineAmount) }
        
        // 调用其他函数
        doSwap();
    }
    
    function doSwap() internal {
        int value;
        // 读取：可以读取到 createPool 中存储的值
        assembly { value := tload(0) }
        
        // value 现在等于 premineAmount
    }
    
    // 交易结束后，tstore 的数据自动清空
}
```

**实际应用场景（PositionManager.sol）：**

**存储时（第292行）：**
```solidity
if (_params.premineAmount != 0) {
    int premineAmount = _params.premineAmount.toInt256();
    assembly { tstore(poolId, premineAmount) }  // 存储预挖数量
}
```

**读取时（第459行）：**
```solidity
int premineAmount = _tload(PoolId.unwrap(poolId));  // 读取预挖数量
if (premineAmount != 0 && _params.amountSpecified == premineAmount) {
    emit PoolPremine(poolId, premineAmount);
}
```

**特点：**
- ✅ 可以跨函数传递（同一交易内）
- ✅ Gas 消耗极低
- ✅ 自动清理，无需手动删除
- ✅ 安全：只在当前交易内可见
- ❌ 交易结束后数据丢失

---

### 3️⃣ Memory（内存存储）

**类比：函数局部变量**
- 仅在函数执行期间存在
- 函数结束后自动销毁
- 无法跨函数传递

```solidity
contract Example {
    function processData(uint[] calldata data) external {
        // memory: 创建临时数组
        uint[] memory tempArray = new uint[](10);
        tempArray[0] = 100;  // 可以修改
        
        // 可以创建复杂的临时结构
        PoolKey memory poolKey = PoolKey({
            currency0: currency0,
            currency1: currency1,
            fee: fee,
            tickSpacing: tickSpacing,
            hooks: IHooks(address(this))
        });
        
        // 调用其他函数
        anotherFunction();
        
        // tempArray 在这里还能用
        require(tempArray[0] == 100);
        
    } // ← 函数结束，tempArray 和 poolKey 都被销毁
    
    function anotherFunction() internal {
        // ❌ 无法访问上面函数中的 tempArray
        // ❌ 无法访问上面函数中的 poolKey
    }
}
```

**特点：**
- ✅ 适合函数内的临时计算
- ✅ Gas 消耗适中
- ✅ 可以修改
- ❌ 无法跨函数传递
- ❌ 函数结束后销毁

---

### 4️⃣ Calldata（只读参数）

**类比：函数参数（const）**
- 仅在函数执行期间存在
- 只读，无法修改
- 最省 Gas 的方式（直接从交易数据读取）

```solidity
contract Example {
    // external 函数的参数推荐用 calldata
    function processArray(uint[] calldata data) external {
        // calldata: 只读，最省 Gas
        uint first = data[0];  // ✅ 读取
        // data[0] = 100;      // ❌ 不能修改！编译错误
        
        // 如果需要修改，必须复制到 memory
        uint[] memory mutableData = data;
        mutableData[0] = 100;  // ✅ 可以修改
    }
    
    // internal/private 函数用 memory
    function internalProcess(uint[] memory data) internal {
        data[0] = 200;  // ✅ 可以修改
    }
}
```

**特点：**
- ✅ 最省 Gas（不需要复制数据）
- ✅ 适合外部函数参数
- ❌ 只读，无法修改
- ❌ 仅限 external 函数

## 📏 生命周期可视化

```
区块链启动
  │
  ├─ storage: storageValue = 100 ✅ 永久保存
  │
  └─ 交易开始
      │
      ├─ transient storage: tstore(key, 300) ◄─── 整个交易期间有效
      │
      ├─ 函数A调用
      │   ├─ calldata: 参数 data ◄─── 只在函数A内有效（只读）
      │   ├─ memory: tempArray 创建 ◄─── 只在函数A内有效
      │   └─ 函数A结束 → memory 销毁 ❌
      │
      ├─ 函数B调用（同一交易内）
      │   ├─ memory: result 创建 ◄─── 只在函数B内有效
      │   ├─ tload(key) ✅ 可以读取到 300
      │   └─ 函数B结束 → memory 销毁 ❌
      │
      └─ 交易结束 → transient storage 清空 ❌
                 → storage 依然保留 ✅ storageValue 还是 100

程序重启/节点重启
  └─ storage: storageValue 依然是 100 ✅
```

## 💡 完整代码示例

```solidity
contract ComprehensiveStorageExample {
    // ========== STORAGE ==========
    uint public storageValue;  // 永久存储
    mapping(PoolId => uint) public flaunchesAt;
    
    // ========== TRANSIENT STORAGE 辅助函数 ==========
    function _tload(bytes32 _key) internal view returns (int value_) {
        assembly { value_ := tload(_key) }
    }
    
    function _tstore(bytes32 _key, int _value) internal {
        assembly { tstore(_key, _value) }
    }
    
    // ========== 综合示例 ==========
    function comprehensiveExample(
        uint[] calldata data  // CALLDATA: 只读参数
    ) external returns (uint) {
        
        // === STORAGE 操作 ===
        storageValue = 100;  // 写入永久存储（贵！）
        
        // === TRANSIENT STORAGE 操作 ===
        bytes32 key = bytes32(uint(1));
        _tstore(key, 200);  // 存储：交易期间有效
        
        // === MEMORY 操作 ===
        uint[] memory tempArray = new uint[](10);  // 创建临时数组
        tempArray[0] = data[0];  // 从 calldata 复制到 memory
        
        // === CALLDATA 操作 ===
        uint firstElement = data[0];  // 读取（最省 Gas）
        // data[0] = 999;  // ❌ 错误！calldata 只读
        
        // 调用另一个函数
        uint result = anotherFunction(tempArray);
        
        return result;
        
    } // ← memory 的 tempArray 销毁
    
    function anotherFunction(
        uint[] memory arr  // MEMORY: 可修改的参数
    ) internal returns (uint) {
        
        // ✅ 可以读取 storage
        require(storageValue == 100);
        
        // ✅ 可以读取 transient storage
        bytes32 key = bytes32(uint(1));
        int tsValue = _tload(key);
        require(tsValue == 200);
        
        // ✅ 可以修改 memory 参数
        arr[0] = 999;
        
        // ❌ 无法访问上个函数的 tempArray（已销毁）
        // ❌ 无法访问上个函数的 calldata data（不在作用域内）
        
        return arr[0];
    }
    
} // ← 交易结束：
  //   - transient storage 清空 ❌
  //   - memory 全部销毁 ❌
  //   - storage 依然保留 ✅
```

## 🎯 如何选择使用哪种存储？

### 使用 Storage 当：
- ✅ 需要永久保存数据
- ✅ 需要跨交易访问数据
- ✅ 合约状态变量
- 💡 例如：余额、所有权、配置信息

### 使用 Transient Storage 当：
- ✅ 需要在同一交易的多个函数间传递数据
- ✅ 需要防止重入攻击（重入锁）
- ✅ 临时标记/验证（如预挖验证）
- ✅ 不需要永久保存的临时状态
- 💡 例如：交易内的临时标记、跨 hook 的数据传递

### 使用 Memory 当：
- ✅ 函数内的临时计算
- ✅ 构建复杂的临时数据结构
- ✅ 需要修改数据副本
- 💡 例如：临时数组、结构体实例、计算中间结果

### 使用 Calldata 当：
- ✅ External 函数参数
- ✅ 只需要读取，不需要修改
- ✅ 想节省 Gas
- 💡 例如：接收大数组参数、批量操作的输入

## 💰 Gas 消耗对比（近似值）

| 操作 | Gas 消耗 | 备注 |
|------|---------|------|
| Storage 首次写入 (SSTORE) | ~20,000 | 从 0 写入非 0 值 |
| Storage 修改 (SSTORE) | ~5,000 | 修改已有的非 0 值 |
| Storage 读取 (SLOAD) | ~2,100 | - |
| Transient Store (TSTORE) | ~100 | EIP-1153 |
| Transient Load (TLOAD) | ~100 | EIP-1153 |
| Memory 扩展 | ~3-6/word | 随内存使用增长 |
| Calldata 读取 | ~16/byte | 最便宜 |

## 📚 实际项目中的应用（PositionManager.sol）

### 场景：预挖（Premine）验证

**需求：** 在创建池子的交易中，允许创建者进行一次预挖操作，但只能在同一交易内进行。

**为什么选择 Transient Storage？**

1. **不能用 memory**：
   - ❌ 无法在 `createPool()` 和 `beforeSwap()` 之间传递
   - 这两个是不同的函数调用

2. **不应该用 storage**：
   - ❌ 太贵（写入 ~20000 gas，删除还需要退款处理）
   - ❌ 需要手动清理 `delete premineAmounts[poolId]`
   - ❌ 会留下永久存储痕迹
   - ❌ 可能被其他交易读取（安全风险）

3. **完美使用 transient storage**：
   - ✅ 可以跨函数传递（同一交易内）
   - ✅ 便宜（~100 gas）
   - ✅ 自动清理（交易结束自动清空）
   - ✅ 安全（只在当前交易可见）

**实现代码：**

```solidity
// 在创建池子时存储
if (_params.premineAmount != 0) {
    int premineAmount = _params.premineAmount.toInt256();
    assembly { tstore(poolId, premineAmount) }  // 💾 存储
}

// 在 swap hook 中验证
int premineAmount = _tload(PoolId.unwrap(poolId));  // 🔍 读取
if (premineAmount != 0 && _params.amountSpecified == premineAmount) {
    emit PoolPremine(poolId, premineAmount);  // ✅ 验证通过
} else {
    // 不是预挖操作，检查是否达到启动时间
    if (_flaunchesAt > block.timestamp) {
        revert TokenNotFlaunched(_flaunchesAt);
    }
}
// 交易结束后，premineAmount 自动清空 🗑️
```

## 📖 总结

| 问题 | 答案 |
|------|------|
| 需要永久保存？ | → Storage |
| 需要在交易内跨函数传递？ | → Transient Storage |
| 只在单个函数内使用？ | → Memory |
| 外部函数参数且只读？ | → Calldata |

**记忆口诀：**
- 💾 **Storage** = 数据库（永久）
- 🔄 **Transient** = Redis（交易级缓存）
- 🧠 **Memory** = 局部变量（函数级）
- 📨 **Calldata** = 只读参数（最省 Gas）
